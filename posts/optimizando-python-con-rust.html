Optimizando Python con Rust: Un Caso Práctico

16 de Junio, 2025 • Por Omar

Python es fantástico por su simplicidad y su ecosistema, pero a veces su rendimiento en tareas computacionalmente intensivas deja que desear. Cuando te enfrentas a un cuello de botella, una solución es reescribir esa parte crítica en un lenguaje de más bajo nivel como C o Rust.

¿Por qué Rust?

Rust ofrece un rendimiento comparable al de C/C++, pero con garantías de seguridad de memoria que eliminan toda una categoría de bugs. Gracias a librerías como PyO3, integrar código Rust en Python es sorprendentemente sencillo.

La combinación de la facilidad de Python para el desarrollo rápido y el rendimiento bruto de Rust para las partes críticas es una estrategia ganadora.

En este análisis, tomamos un script que procesa imágenes pixel por pixel y medimos el tiempo de ejecución. Luego, aislamos la función de procesamiento y la reescribimos en Rust.

El Código

Así se ve la función en Rust que se expondrá a Python:


use pyo3::prelude::*;

/// Procesa una imagen para invertir sus colores.
#[pyfunction]
fn process_image_fast(mut image_data: Vec) -> PyResult> {
    for pixel in image_data.chunks_mut(4) {
        pixel[0] = 255 - pixel[0]; // R
        pixel[1] = 255 - pixel[1]; // G
        pixel[2] = 255 - pixel[2]; // B
    }
    Ok(image_data)
}

El resultado fue una aceleración de más de 100 veces en la ejecución de la tarea. Esto demuestra el poder de usar la herramienta adecuada para cada trabajo, combinando lo mejor de ambos mundos.
